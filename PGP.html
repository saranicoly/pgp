Fluxo de Migração de Dados: SUAP → SIADS

Ilustração interativa do processo de extração, padronização e migração dos dados patrimoniais dos IFPE


SISTEMA ORIGEM
1

Sistema SUAP (Sistema Unificado de Administração Pública)

O SUAP é um sistema antigo utilizado para gestão administrativa que contém todos os dados patrimoniais dos Institutos Federais de Educação, Ciência e Tecnologia (IFPE).

2

Processamento com Tabula.technology

O Tabula é uma ferramenta online que extrai tabelas de arquivos PDF e as converte para formatos estruturados como CSV, facilitando a manipulação dos dados.

Tabula.technology
Ferramenta web para extração de tabelas de PDFs
Clique para acessar a ferramenta

Processo executado:

  • Upload do PDF gerado pelo SUAP
  • Seleção automática ou manual das tabelas no documento
  • Extração dos dados tabulares para formato estruturado
  • Exportação para arquivo CSV (Comma-Separated Values)

Resultado: Dados padronizados em formato CSV, pronto para a próxima etapa de refinamento.

3

Refinamento com OpenRefine

OpenRefine é uma ferramenta open-source para limpeza e transformação de dados. Ela roda localmente, garantindo a privacidade dos dados patrimoniais, e utiliza algoritmos de clustering para identificar e unificar dados similares.

OpenRefine
Ferramenta open-source para limpeza e transformação de dados
Clique para acessar o site oficial

Operações realizadas:

  • Importação do arquivo CSV gerado pelo Tabula
  • Identificação e correção de inconsistências nos dados
  • Clustering de dados para agrupar itens similares (ex: "cadeira", "cadeira ergonômica", "cadeira giratória")
  • Padronização de formatos, unidades de medida e categorias
  • Exportação dos dados limpos e estruturados

Vantagens: Processamento local (garante privacidade), algoritmos inteligentes de clustering, transformações em lote.

4

Padronização Inteligente com LM Studio

Após o processamento inicial no OpenRefine, utilizamos o LM Studio com modelos de linguagem locais para uma padronização inteligente das descrições patrimoniais, mantendo a privacidade dos dados.

LM Studio
Plataforma para executar modelos de linguagem localmente
Clique para acessar o site oficial

Processo executado:

  • Extrair apenas a lista de descrições únicas do arquivo processado pelo OpenRefine
  • Passar essa lista para o LM Studio com um prompt específico para padronização
  • Utilizar modelos de linguagem locais para garantir a privacidade dos dados
  • Gerar uma tabela padronizada com mapeamento original → padronizado

Prompt utilizado no LM Studio:

"Tenho uma lista de descrições de patrimônio (Coluna A). Preciso que você gere uma Coluna B com o 'Nome Padronizado'.

Regras de Ouro:

  • CORRIGIR: Erros de português, abreviações ('Cad.' → 'Cadeira') e formatação.
  • MANTER: A especificidade do item. 'Cadeira de Escritório' é diferente de 'Cadeira Escolar'. Mantenha essa diferença. Não generalize para apenas 'Cadeira'.
  • FORMATO: Me entregue APENAS uma tabela CSV com duas colunas: original; padronizado.

Lista: [Cole sua lista de únicos aqui]"

Exemplo de transformação:

Cad. ergonomica → Cadeira Ergonômica
mesa escritorio → Mesa de Escritório
quadro branco → Quadro Branco
computador desktop → Computador Desktop

Vantagens: Processamento local com IA (garante privacidade), correção inteligente de inconsistências, padronização semântica mantendo especificidades, redução de trabalho manual.

Dados Sensíveis Protegidos
O LM Studio roda 100% localmente, garantindo que dados patrimoniais sensíveis nunca saiam da rede interna.

SISTEMA DESTINO
5

Sistema SIADS (Sistema Integrado de Administração de Serviços)

O SIADS é o novo sistema do governo que substituirá o SUAP, oferecendo melhor gestão, interface intuitiva e recursos avançados para administração dos bens patrimoniais.

SIADS
Sistema moderno de gestão patrimonial
Clique para acessar o sistema

Processo de migração:

  • Importação dos dados refinados do OpenRefine
  • Mapeamento dos campos entre os sistemas (SUAP → SIADS)
  • Validação final dos dados migrados
  • Ativação do sistema SIADS para uso operacional

Resultado final: Todos os dados patrimoniais dos IFPEs migrados para um sistema moderno, com dados padronizados, limpos e prontos para uma gestão eficiente.
